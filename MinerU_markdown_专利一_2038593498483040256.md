# (54) 发明名称

一种基于多模态传感器融合的工业设备实时健康监测系统

# (57)摘要

本发明提供一种基于多模态传感器融合的工业设备实时健康监测系统，包括采集设备运行参数的多模态传感器模块，对传感器数据进行清洗、预处理、特征提取和初步异常检测的边缘计算网关，接收来自边缘计算网关发布的数据并转发的MQTT数据采集与传输模块，接收MQTT转发的海量实时数据流的Kafka实时流处理模块，将Kafka实时流处理模块传输的数据形成统一多维特征向量的数据融合与特征工程模块，分别利用融合后的多模态特征数据进行时序预测的LSTM时间预测模块、进行分类诊断的随机森林分类模块、进行异常行为识别的自编码器异常检测模块，通过实时数据流进行可视化展示与交互式查询的数字孪生模块。本系统能实现对工业设备运行状态全面精准评估和故障预警。

多模态传感器模块：部署在工业设备上，用于实时采集包括振动信号、温度信号、声学信号在内的多模态设备运行参数；

边缘计算网关：用于接收多模态传感器数据，对数据进行数据清洗、预处理、特征提取和初步异常检测，并对处理的数据进行压缩；

MQTT数据采集与传输模块：利用MQTT协议的轻量级、发布/订阅模式，接收来自边缘计算网关发布的传感器数据并转发，实现发布者与订阅者的解耦；

Kafka实时流处理模块：用于接收MQTT数据采集与传输模块转发的海量实时数据流，进行数据的缓冲、分发、实时聚合和清洗，为后续提供高质量的输入数据；

LSTM时间预测模块：利用LSTM基于来自数据融合与特征工程模块融合后的多模态时序特征数据，对设备的健康指标进行时序预测，提前预警潜在的性能衰退或故障趋势；

随机森林分类模块：通过包含来自数据融合与特征工程模块融合后的多模态特征向量以及对应的时序特征状态向量，利用随机森林算法对设备运行状态或故障类型进行分类诊断，提高故障识别的准确性；

自编码器异常检测模块：利用自编码器对设备运行数据进行无监督学习，构建正常运行模式的特征表示，当包含来自数据融合与特征工程模块融合后的多模态特征向量异常数据输入时，通过自编码器的重构误差识别出偏离正常模式的异常行为，实现早期故障的无监督检测；

数字孪生模块：基于工业设备的几何、物理、行为参数，构建工业设备的虚拟数字孪生模型，并通过来自Kafka实时流处理模块、LSTM时序预测模块、随机森林分类模块和自编码器异常检测模块的实时数据流驱动虚拟数字孪生模型更新，实现设备状态的可视化展示与交互式查询。

# 一种基于多模态传感器融合的工业设备实时健康监测系统

# 技术领域

[0001] 本发明涉及工业设备监测技术领域,具体涉及一种基于多模态传感器融合的工业设备实时健康监测系统。

# 背景技术

[0002] 工业设备的智能化和自动化水平不断提高。然而，设备故障仍然是影响生产效率、增加运营成本甚至引发安全事故的重要因素。传统的设备维护方式，如定期维护或事后维护，往往效率低下，无法有效避免突发故障，也未能充分利用设备的剩余寿命。近年来，基于状态的维护(Condition-Based Maintenance, CBM)逐渐成为主流，其核心是通过监测设备运行参数来评估设备健康状况。目前，CBM系统通常依赖单一类型的传感器数据（如振动数据）进行分析，这在一定程度上限制了对设备复杂故障模式的识别能力。例如，某些故障可能在振动信号不明显时，已在温度或声学特征上有所体现。此外，现有系统在数据传输、实时处理和高级分析方面仍存在挑战，例如，数据异构性与融合困难：工业设备运行数据来源多样，包括振动、温度、电流、声学等，这些数据类型、采样频率、数据格式各异，难以有效融合进行综合分析。数据传输效率与实时性：大规模工业设备产生的海量数据对传统网络带宽和传输协议构成挑战，难以保证数据传输的实时性和可靠性。边缘计算能力不足：传统系统多依赖云端进行所有数据处理，导致数据传输延迟高、网络负载大，且对网络连接的稳定性要求高。高级分析模型欠缺：多数系统采用基于阈值或简单统计的分析方法，难以捕捉设备运行的复杂非线性特征和微弱异常信号，导致误报或漏报。缺乏可视化与交互：缺乏直观的设备健康状态可视化界面和有效的数字孪生模型，使得操作人员难以全面了解设备运行状况并进行有效干预。

[0004] 为了解决上述技术问题,本发明采用了如下的技术方案:

[0007] 边缘计算网关:用于接收多模态传感器数据,对数据进行数据清洗、预处理、特征提取和初步异常检测,并对处理的数据进行压缩;

[0010] 数据融合与特征工程模块：用于对来自Kafka实时流处理模块的不同模态传感器数据进行时间同步、空间对齐、特征提取和特征融合，形成统一的多维特征向量；

[0013] 自编码器异常检测模块：利用自编码器对设备运行数据进行无监督学习，构建正常运行模式的特征表示，当包含来自数据融合与特征工程模块融合后的多模态特征向量异常数据输入时，通过自编码器的重构误差识别出偏离正常模式的异常行为，实现早期故障的无监督检测；

[0041] 作为具体实施例,所述多模态传感器模块采用多种类型的传感器协同工作,以获取设备运行的全面信息,具体包括:用于采集设备在不同方向上的振动信号的振动传感器(如加速度计),反映设备的机械运行状态,如轴承磨损、不平衡、不对中等;用于采集设备关键部件(如电机、轴承、齿轮箱)温度变化的温度传感器(如热电偶、热电阻),反映设备的散热状况和摩擦发热情况;以及用于采集设备运行时的声学信号的声学传感器(如麦克风),通过声谱分析可以识别异常噪音,如异响、摩擦声等。当然,本领域技术人员可以根据具体设备需求,还可以集成其他传感器如电流传感器、压力传感器、流量传感器等,以获取更全面的设备运行参数。

[0042] 作为具体实施例,所述多模态传感器模块中的各个传感器以高采样率实时采集数据,并通过有线(如Ethernet、RS485)或无线(Wi-Fi、ZigBee、LoRa)的方式将数据传输至边缘计算网关。

[0043] 作为具体实施例, 所述边缘计算网关是本系统的关键组成部分, 它承担了数据预处理和部分智能分析任务, 提高了响应速度。主要功能包括数据采集与汇聚: 接收来自多个多模态传感器的原始数据; 数据清洗与预处理: 对原始数据进行去噪、滤波、缺失值填充和数据格式转换等操作, 保证数据质量; 特征提取: 在边缘侧对原始数据进行初步的特征提取, 例如对振动信号进行时域 (均方根、峰值、峭度等)、频域 (频谱、功率谱密度) 特征提取, 对温度信号进行趋势分析, 对声学信号进行声谱特征提取; 初步异常检测: 对于一些简单的异常模式, 可以在边缘侧进行实时检测, 并触发本地警告, 实现快速响应; 数据压缩与传输: 对处理后的数据进行压缩, 并通过MQTT协议发布到MQTT数据采集与传输模块。边缘计算网关通常采用具备一定计算能力和存储能力的工业级计算机或嵌入式设备来实现, 如工业PC、树莓派等。

[0044] 作为具体实施例,所述MQTT数据采集与传输模块中,MQTT (Message Queuing Telemetry Transport) 是一种轻量级的消息发布/订阅协议,非常适合在低带宽、高延迟或不可靠网络环境下进行物联网数据传输。包括发布者(Publisher):边缘计算网关作为MQTT 发布者,将预处理和特征提取后的设备健康数据发布到特定的MQTT主题(Topic)上;订阅者(Subscriber):云端平台上的Kafka实时流处理模块作为MQTT订阅者,订阅相关主题,实时接收来自所有边缘网关的设备数据;MQTT Broker:位于云端或独立的服务器上,负责接收发布者的消息并将其转发给所有订阅者,实现发布者与订阅者的解耦。MQTT的优势在于其低功耗、低带宽占用、支持QoS(服务质量)等级以及发布/订阅模式,能够有效管理大规模传感器数据流。

[0045] 作为具体实施例, Apache Kafka 是一个分布式流处理平台, 具有高吞吐量、低延迟、高可用性和可扩展性等特点, 非常适合处理工业物联网产生的海量实时数据。所述 Kafka 实时流处理模块的主要功能包括数据摄入: Kafka 作为数据管道的入口, 从 MQTT 订阅者接收所有设备数据流, 并将其存储在不同的主题 (Topic) 中; 数据缓冲与分发: Kafka 能够有效缓冲瞬时高峰数据, 并根据不同的下游消费者需求, 将数据分发到不同的处理单元; 实时聚合和清洗: 利用 Kafka Streams 或 KSQL 等组件, 对数据进行实时的聚合、过滤、转换和清洗, 为后续算法提供高质量的输入数据; 容错与持久化: Kafka 的分布式特性和数据复制机制保证了数据处理的容错性和持久性。Kafka 构建了一个强大的实时数据骨干网, 确保了数据从边缘到云端的顺畅流动和高效处理。

[0046] 作为具体实施例，所述数据融合与特征工程模块是多模态传感器数据价值挖掘的关键，主要功能包括时间同步：确保来自不同传感器的异构数据在时间维度上对齐，可以通过现有的统一时间戳、插值或重采样方法来实现；空间对齐：如果传感器部署在设备的不同位置，需要考虑其空间关系，以便后续分析，可以通过现有的空间对齐来完成；特征提取包括：时域特征，如均值、方差、均方根(RMS)、峰值、峰峰值、峭度、偏度、波形因子、脉冲因子等；频域特征，如傅里叶变换(FFT)后的幅值谱、功率谱密度(PSD)、主频、边频带能量等；时频域特征，如小波变换、短时傅里叶变换等；统计特征：如互相关、互信息等，用于衡量不同模态数据之间的关联性；特征融合：将提取出的多模态特征进行融合，形成一个多维的特征向量，融合方式可以是现有简单的特征拼接，也可以是更复杂的深度学习融合方法。

[0047] 作为具体实施例,所述LSTM时间预测模块中的LSTM(Long Short-Term Memory,长短期记忆网络)是一种特殊的循环神经网络(RNN),擅长处理和预测时间序列数据,能够有效捕捉时间序列中的长期依赖关系。其输入来自:数据融合与特征工程模块融合后的多模态时序特征数据。网络模型训练:使用历史正常运行数据训练LSTM模型,学习设备健康指标(如振动RMS、温度等)的时序变化规律。网络模型预测:基于当前和历史数据,LSTM模型预测设备未来一段时间内的健康指标值。趋势预警:将预测值与预设阈值进行比较,如果预测值超出安全范围或呈现恶化趋势,则发出预警。LSTM能够有效预测设备性能衰退趋势,实现早期预警,为预测性维护提供时间窗口。

[0048] 作为具体实施例, 所述随机森林分类模块中的随机森林 (Random Forest) 是一种集成学习算法, 通过构建多棵决策树并综合其预测结果, 具有高准确性、鲁棒性强、不易过拟合等优点。其输入来自: 数据融合与特征工程模块融合后的多模态特征向量以及对应的设备故障标签 (如正常、轴承磨损、齿轮损坏、不平衡等)。模型训练: 使用带有故障标签的历史数据训练随机森林模型, 学习不同故障模式下的特征分布规律。故障分类: 当有新的实时特征数据输入时, 随机森林模型能够将其归类到相应的故障类型中, 实现故障诊断。特征重要性评估: 随机森林还能评估各个特征对故障分类的贡献度, 有助于工程师理解故障机制。随机森林在多模态数据融合后的故障分类任务中表现出色。

[0049] 作为具体实施例, 所述自编码器异常检测模块中的自编码器 (AutoEncoder) 是一种无监督神经网络, 通过学习输入数据的压缩表示 (编码) 和从压缩表示中重构原始数据 (解码), 来发现数据中的模式。网络模型训练: 仅使用大量正常运行状态下的设备数据训练自编码器, 网络模型会学习并重构正常数据的内在模式。异常识别: 当输入异常数据时, 自编码器的重构误差 (即输入数据与重构数据之间的差异) 会显著增大, 因为模型未在异常数

据上进行训练,无法很好地重构。阈值设定:通过对正常数据的重构误差进行统计分析,设定一个异常阈值,当实时数据的重构误差超过该阈值时判定为异常,以此实现通过自编码器的重构误差识别出偏离正常模式的异常行。自编码器能够有效识别出未知的、新出现的异常模式,实现早期故障的无监督检测。

[0050] 作为具体实施例,所述数字孪生模块中的数字孪生(Digital Twin)是物理实体的虚拟映射,通过实时数据驱动,实现物理实体在虚拟空间中的动态、高保真再现。具体地,孪生模型构建:是基于工业设备的几何、物理、行为参数构建的虚拟数字孪生模型。据驱动与同步:来自Kafka实时流处理模块、LSTM时序预测模块、随机森林分类模块和自编码器异常检测模块将处理后的设备实时数据流传输给虚拟数字孪生模型,驱动数字孪生模型的更新,使其状态与物理设备保持高度同步。可视化展示与交互式查询:数字孪生模型提供三维可视化界面,直观展示设备的实时运行状态、健康指标、故障位置和类型,用户可以通过数字孪生模型进行交互式查询、模拟仿真和故障追溯。预测与优化:结合算法层(包括数据融合与特征工程模块、LSTM时间预测模块、随机森林分类模块和自编码器异常检测模块)的预测结果,数字孪生可以模拟设备在不同工况下的未来健康趋势,为预测性维护和运行优化提供决策支持。数字孪生技术为设备健康管理提供了沉浸式、全生命周期的管理视角。

[0051] 作为具体实施例，本监测系统还会根据分析结果生成告警信息，并通过短信、邮件、APP推送等方式通知相关人员。维护人员根据系统提供的健康报告和预测信息，制定预测性维护策略。

[0052] 为了更好理解本发明提供的基于多模态传感器融合的工业设备实时健康监测系统，以下将结合具体实施方式来进行说明：

[0057] S5、算法分析：训练LSTM网络模型预测未来24小时的风机轴承温度和振动RMS值，如果预测值超过安全阈值，系统发出“轴承温度过高预警”或“振动异常趋势预警”。基于历史故障数据(如轴承磨损、叶片不平衡、电机绕组故障等)，训练随机森林模型，当新的特征数据输入时，模型识别出当前故障类型，例如“轴承磨损(初期)”。使用风机正常运行数据训练自编码器，当风机出现异常噪音(如叶片刮擦声)时，声学特征的重构误差会显著增大，系统识别出“未知异常噪音告警”。

[0058] 6、数字孪生：在数字孪生平台上，风机的三维模型会实时显示其振动幅值、温度分布和声学频谱图。当系统发出告警时，数字孪生模型上对应的部件会高亮显示，并弹出详细的故障信息和建议维护方案。

[0059] 7、维护决策：维护人员通过数字孪生界面或手机APP接收告警，并根据系统提供的预测和诊断信息，提前安排检修计划，避免风机突发停机。通过上述实施方式，本系统能够
